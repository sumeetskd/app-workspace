Attribute Binding will directly use the values using Interpolation

public disableButton = true
< button disabled='{{disableButton}}' >

Property Binding

public disableButton = true
Use [ ] to dynamically bind the attributes with the property.
[disabled] = 'disableButton'
NOTE: When the attribute starts accepting dynamic values. It is no loger stays static. So, it becomes a property.


Every HTML attirbute is not having a relative property to handle dynamically. You have to use attribute as property binding with "attr" as prefix.
Syntax:
[attr.attributeName]="dynamicReference"

Eg: attr.height is used instead of the height attribute: [width]="tableWidth" [attr.height]="tableHeight"

Two Way Binding

For two way binding we need:

  1. The data by using property binding: [ ]
  2. It uses the Event to compare and update the data-value that we have fetched using property binding in the view/html-template
Event Binding: (click) = "showAlert()"

In MVC Framework, Model is having the value. To accesss the value we have to use ngModel which will use both property and event binding. Hence, it is called 2 way binding.
Syntax = [(ngModel)]

Breakdown of [(ngModle)]

  • [ ] --> Represents Property Binding
  • ( ) --> Represents Event Binding. ie. (change) event is being used to compare the values.

NOTE: ngModel is also called as a Directive. As it has the ability to change the html elements dynamically.

ngModel is not available in the @angular/core library, to use ngModel we have to import FormsModule from '@angular/forms'
Add FormModel in the imports: array metadeta.

  • ngModel is a directive responsible for handling two-way binding.
  • It uses "Property and Event" binding Technique.
  • Property Binding is used to bind the model value.
  • Event Binding is used to update the changes.
  • [ ] defines property binding.
  • ( ) defines event binding.
  • [(ngModel)] two-way binding.
  • ngModel is a member of "FormModel" present in "@angular/forms" library

Directives

  • Directive is technically a function
  • Directive is responsible for converting the static DOM element into dynamic DOM
  • Directive makes HTML more declarative.
  • It extends HTML.
    < input type="text" [(ngModel)] >
  • Directive have different types of functionalities, they can be used as:
    • Element:

      Directive are used as elements to return markup.
      < ng-form > < /ng-form >> < router-outlet > < /router-outlet >>

    • Attribute:

      Directives are used as attributes to extend HTML.
      < input type="text" ngModel #txtName="ngModel" >

    • Class:

      Directives are used as Classes to make the markup more interactive. < span class="ng-valid"> < /span>
      < span class="ng-pristine"> < /span>

    • Comment:

      Directives are used as Comments to target legacy browser.
      < !-- ngModel:txtname --> *ignore additional space between tags

Angular Directive

Angular Directive are classified into 3 types:

  • Component Directives
  • Structural Directives
  • Attribute Directives

Component Directives

  • The component directive is the most common directive in Angular.
  • It is a template that comprises of logic, presentation and styles.
  • It is used for dynamically rendering HTML and handling interaction with users.
Ex.:
  • logic.component.ts -> logic
  • logic.component.html -> html
  • logic.component.style -> style
Selector for component directive: < app-comp > < /app-comp >
- We can access and use built-in components from a library called "Angular Material"

Structural Directive

  • A structural directive is responsible for changing the DOM structure dynamically.
  • It can add or remove element, It can iterate over element, it can switch between elements etc.
  • Angular Structural directives are:
    1. ngIf
    2. ngSwitch
    3. ngFor
  • Structural Directives are added to HTML DOM element by using '*'
    • < div *ngIf="" >
    • < div *ngFor="" >
  • Every HTML element can use only one structural directive.
    • < li *ngFor="" *ngIf=""> => Invalid, only one structural directive is allowed

    NgIf Directive(ngIf)

    • It is a structural directive used to add or remove any DOM elemet dynamically.
    • It uses a boolean condition or value to add or remove DOM element.
    Hide an Image:
    < img style="display:none">

    This will keep the image in your page DOM but hides in the UI. It will not reduce burden on Page.

    Syntax:
    < img *ngIf="booleanValue/Expression">
